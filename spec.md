# CookPlan Specification

Every change must be represented here as a user story before implementation begins.

---

## User Stories

### US-001 – View this week's cooking schedule
**As a** user,
**I want to** see the cooking assignments for each day of the current week,
**So that** I know who is cooking on which day.

### US-002 – Add a person to the roster
**As a** user,
**I want to** add a person (name + colour) to the cooking roster,
**So that** they can be assigned to cook.

### US-003 – Remove a person from the roster
**As a** user,
**I want to** remove a person from the roster,
**So that** they no longer appear in assignment suggestions.

### US-004 – Assign a person to cook on a specific day
**As a** user,
**I want to** assign a person from the roster to a specific date,
**So that** the schedule reflects who is responsible for cooking that day.

### US-005 – Attach a meal idea to an assignment
**As a** user,
**I want to** choose (or type) a meal when creating an assignment,
**So that** the schedule shows what will be cooked, not just who cooks.

### US-006 – Mark an assignment as completed
**As a** user,
**I want to** mark a day's assignment as done,
**So that** the schedule visually distinguishes past from upcoming cooking.

### US-007 – Add a meal idea to the meal list
**As a** user,
**I want to** add a named meal idea with an optional description and tags,
**So that** it can be quickly selected when creating future assignments.

### US-008 – Remove a meal idea from the meal list
**As a** user,
**I want to** remove a meal idea I no longer want,
**So that** the list stays relevant.

### US-009 – Navigate between schedule, people, and meals
**As a** user,
**I want to** switch between the Schedule, People, and Meals views via a navigation bar,
**So that** I can manage all aspects of the app from one place.
